# <a name="sequence-of-operations-for-building-mfc-applications"></a>MFC アプリケーションの作成手順

次の表では、MFC アプリケーションを開発するときに従うが通常の一般的な順序について説明します。

### <a name="sequence-for-building-an-application-with-the-framework"></a>フレームワークとアプリケーションを構築するためのシーケンス

|タスク|そうですよね|フレームワークを動作します。|
|----------|------------|------------------------|
|スケルトン アプリケーションを作成します。|実行、 [MFC アプリケーション ウィザード](../mfc/reference/mfc-application-wizard.md)します。 [オプション] ページで、目的のオプションを指定します。 オプションにより、アプリケーションの COM コンポーネントをコンテナー、またはその両方があります。Automation; を追加します。により、アプリケーションをデータベースに注意してください。|MFC アプリケーション ウィザードには、スケルトン アプリケーションの場合、アプリケーション、ドキュメント、ビュー、およびウィンドウ フレームのソース ファイルを含むファイルが作成されます。リソース ファイル。は、プロジェクト ファイル仕様に合わせてすべて、他のユーザー。|
|どのようなフレームワークおよび MFC アプリケーション ウィザードが、独自のコードの行を追加することがなく提供を参照してください。|スケルトン アプリケーションを構築し、Visual C で実行します。|スケルトン アプリケーションは、多くの標準の派生**ファイル**、**編集**、**ビュー**、および**ヘルプ**フレームワークからのメニュー コマンド。 MDI アプリケーションも完全に機能の Windows メニューを取得し、フレームワークは、作成、配置、および MDI 子ウィンドウの破棄を管理します。|
|アプリケーションのユーザー インターフェイスを構築します。|Visual C を使用して、[リソース エディター](../windows/resource-editors.md)アプリケーションのユーザー インターフェイスを視覚的に編集します。<br /><br /> -メニューを作成します。<br />-アクセラレータを定義します。<br />-ダイアログ ボックスを作成します。<br />-作成し、ビットマップ、アイコン、およびカーソルを編集します。<br />-編集ツールバーの MFC アプリケーション ウィザードによって作成します。<br />-作成し、その他のリソースを編集します。<br /><br /> ダイアログ エディターのダイアログ ボックスをテストすることもできます。|MFC アプリケーション ウィザードによって作成された既定のリソース ファイルでは、必要なリソースを多数提供します。 Visual C では、既存のリソースを編集し、簡単かつ視覚的には、新しいリソースを追加することができます。|
|メニューをハンドラー関数にマップします。|使用して、**イベント**ボタン、[プロパティ ウィンドウ](/visualstudio/ide/reference/properties-window)メニューとアクセラレータをコード内のハンドラー関数に接続します。|[プロパティ] ウィンドウは、ソース ファイルを指定し、多くの手動コーディング作業の管理に、メッセージ マップ エントリと空の関数テンプレートを挿入します。|
|ハンドラー コードを記述します。|クラス ビューを使用すると、ソース コード エディターでコードに直接移動できます。 ハンドラー関数のコードを入力します。 クラス ビューを使用して、プロジェクトにコードを追加するウィザードについての詳細については、次を参照してください。[コード ウィザードによる機能の追加](../ide/adding-functionality-with-code-wizards-cpp.md)します。|クラス ビューは、エディターが開き、空の関数テンプレートまでスクロールし、カーソルが置かれます。|
|コマンド ツール バー ボタンにマップします。|メニューやアクセラレータ コマンドに該当するコマンド ID を割り当てることにより、ツールバーの各ボタンにマップします。|フレームワークは、描画、有効にすると、無効にすると、チェック、およびツール バー ボタンの他の視覚的な側面を制御します。|
|ハンドラー関数をテストします。|プログラムを再構築し、ハンドラーが正常に動作をテストする組み込みのデバッグ ツールを使用します。|ステップまたはハンドラーの呼び出し方法を確認するコードをトレースできます。 ハンドラー コードを入力した場合、ハンドラーはコマンドを実行します。 フレームワークは、メニュー項目と処理されないツール バー ボタンに自動的に無効になります。|
|追加[ ダイアログ ボックス](../mfc/dialog-boxes.md)します。|ダイアログ エディターのダイアログ テンプレート リソースをデザインします。 次に、ダイアログ クラスと、ダイアログ ボックスを処理するコードを作成します。|フレームワークでは、ダイアログ ボックスを管理し、ユーザーが入力した情報の取得を容易にします。|
|初期化、検証、およびダイアログ ボックスのデータを取得します。|ダイアログ ボックスのコントロールの初期化および検証されるの方法を定義することもできます。 Visual Studio を使用して、ダイアログ クラスにメンバー変数を追加し、ダイアログ コントロールにマップします。 データを入力すると、各コントロールに適用される検証規則を指定します。 希望する場合は、独自のカスタム検証を提供します。|フレームワークは、ダイアログ ボックスの初期化と検証を管理します。 無効な情報を入力すると、フレームワークはメッセージ ボックスを表示、データを再入力できるようにします。|
|その他のクラスを作成します。|クラス ビューを使用すると、追加のドキュメント、ビュー、および MFC アプリケーション ウィザードによって自動的に作成されたもの以外のフレーム ウィンドウ クラスを作成できます。 その他のデータベースのレコード セット クラスやダイアログのクラスを作成できます。 (クラス ビューを MFC クラスから派生しないクラスを作成できます)。|クラス ビューでは、ソース ファイルにこれらのクラスを追加し、その接続を処理するコマンドに定義できます。|
|すぐに使用できるコンポーネントをアプリケーションに追加します。|使用して、`New Item dialog box`さまざまなアイテムを追加します。|これらのアイテムは、簡単に、アプリケーションに統合して、多くの作業を保存できます。|
|ドキュメント クラスを実装します。|アプリケーション固有のドキュメント クラスまたはクラスを実装します。 データ構造を保持するメンバー変数を追加します。 データへのインターフェイスを提供するメンバー関数を追加します。|フレームワークは、ドキュメントのデータ ファイルと対話する方法を既に知っています。 これは、ことができますを開くとドキュメント ファイルを閉じます、読み取りおよび書き込み、ドキュメントのデータ、および他のユーザー インターフェイスを処理します。 ドキュメントのデータを操作する方法に集中できます。|
|保存、開くを実装し、コマンドとして保存します。|ドキュメントのコードを記述`Serialize`メンバー関数。|フレームワークのダイアログ ボックスが表示されます、**オープン**、**保存**、および**名前を付けて保存**コマンドを**ファイル**メニュー。 それを書き込み、読み取りで指定されたデータ形式を使用してドキュメントのバックアップ、`Serialize`メンバー関数。|
|ビュー クラスを実装します。|ドキュメントに対応する 1 つまたは複数のビュー クラスを実装します。 クラス ビューによるユーザー インターフェイスにマップしてあるビューのメンバー関数を実装します。 さまざまな[CView](../mfc/reference/cview-class.md)-派生クラスが利用可能[CListView](../mfc/reference/clistview-class.md)と[CTreeView](../mfc/reference/ctreeview-class.md)します。|フレームワークは、ほとんどのドキュメントとビューの間のリレーションシップを管理します。 ビューのメンバー関数へのアクセス、ビューのドキュメントの画面や印刷ページには、そのイメージをレンダリングして、応答を編集コマンドのユーザーのドキュメントのデータ構造を更新します。|
|既定の印刷を強化します。|複数ページ印刷をサポートする必要がある場合は、ビューのメンバー関数をオーバーライドします。|フレームワークは、サポート、**印刷**、**ページ設定**、および**印刷プレビュー**コマンドを**ファイル**メニュー。 ドキュメントを複数のページに分割する方法を指示する必要があります。|
|スクロール機能を追加します。|スクロールをサポートする必要がある場合は、またはから複数のビュー クラスを派生させる[CScrollView](../mfc/reference/cscrollview-class.md)します。|ビューは、ビュー ウィンドウが小さすぎるときに、スクロール バーを自動的に追加します。|
|フォーム ビューを作成します。|ダイアログ テンプレート リソース ビューを基にするには、ビューのクラスまたはクラスから派生させる[CFormView](../mfc/reference/cformview-class.md)します。|ビューは、コントロールを表示するのにダイアログ テンプレート リソースを使用します。 ユーザーは、ビュー内のコントロールにコントロールからタブことができます。|
|データベースのフォームを作成します。|フォーム ベースのデータ アクセス アプリケーションを実行する場合に、派生ビュー クラスから[CRecordView](../mfc/reference/crecordview-class.md) (の ODBC プログラミング)。|フォーム ビューでは、ように機能するビューがそのコントロールがのフィールドに接続されている、 [CRecordset](../mfc/reference/crecordset-class.md)データベース テーブルを表すオブジェクト。 MFC は、コントロールとレコード セットの間のデータを移動します。|
|単純なテキスト エディターを作成します。|単純なテキスト エディターに表示する場合は、派生ビュー クラスまたはクラスから[CEditView](../mfc/reference/ceditview-class.md)または[CRichEditView](../mfc/reference/cricheditview-class.md)します。|関数、クリップボードのサポート、および入力/出力ファイルを編集ビューを提供します。 `CRichEditView` スタイル設定されたテキストを提供します。|
|分割ウィンドウを追加します。|ウィンドウの分割をサポートする場合は、追加、 [CSplitterWnd](../mfc/reference/csplitterwnd-class.md) SDI フレーム ウィンドウまたは MDI 子ウィンドウにオブジェクトし、ウィンドウのフック[OnCreateClient](../mfc/reference/cframewnd-class.md#oncreateclient)メンバー関数。|フレームワークは、スクロール バーの横にある分割ボックス コントロールを提供し、複数のペインに、ビューの分割を管理します。 ウィンドウを分割すると、フレームワークは作成し、ドキュメントに追加のビュー オブジェクトをアタッチします。|
|ビルド、テスト、およびアプリケーションをデバッグします。|ビルド、テスト、およびアプリケーションをデバッグするには、Visual C の機能を使用します。|Visual C では、コンパイル、リンク、およびその他のオプションを調整できます。 ソース コードやクラス構造体を参照することもできます。|
